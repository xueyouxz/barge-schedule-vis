# 一、约束层参数的输入数据变化分析
两类参数性质完全不同：

固定参数（写死在 sim_config 中，不随输入变化）
参数	含义	示例值
MAX_TEU_RATE	最大 TEU 装载率上限	1.0（100%）
MAX_FULL_TEU_RATE	最大满箱率	0.85
TRANSSHIP_COST_TEU	中转费用（按箱型）	小箱 200×3=600元，大箱 300元
COST_BETWEEN_SEAPORTS	宁波各码头间转码头费用矩阵	BLCTZS→BLCT3：150元/箱
travel_time_matrix	各港间航行时间	LTA→TC2：12.5h
left_container_penalty	未装箱惩罚分	-10分/箱
max_berth	最大靠泊次数（命令行参数）	默认 1
动态参数（由输入数据实时推导）

动态参数（由输入数据实时推导）
参数	如何随输入变化
预装池 / 配装池	由每个箱的 ETD、箱就绪时间 与当前仿真时刻对比筛选，输入箱不同 → 进池的箱不同
各箱的 can_transship	由输入字段动态判断：危险品(危类≠空)、箱主为 COSCO/ZGH、急货类别=2 → 强制 False
各船的 barge_direct	由 vessel_info.csv 中的 换装标记 字段决定（每场景不同）
航线候选概率	set_base_prob_by_dest() 根据配装池中各目的港货量分布实时计算，货多的目的港方向被赋予更高概率
采样数 n_sample	由航线概率分布熵值推导，货物分布越集中采样越少
时间约束窗口	中转可行性判断：船0在TC2卸货时刻 vs 船2在TC2开始装货时刻，由输入中各船 实际离港时间 决定
结论：约束边界值（费用、比率、惩罚）是固定的；约束可行域（哪些箱能进池、哪些路径可行、航线采样空间）完全由输入数据动态构建。

# 二、转码头率的定义与计算
转码头 ≠ 中转（两个完全不同的概念）
术语	发生地点	含义
中转（transship）	内河港层面（南京→太仓）	货物从南京港驳船卸到太仓后，换装上另一艘船再赴宁波
转码头（transport）	宁波港区内部	货物卸在宁波某码头后，通过陆运转运到同港区内另一个码头
转码头的业务背景
宁波港有多个相互独立的码头公司，驳船只能靠泊有限几个：

当某个箱的 干线码头（输入字段，货轮靠泊的实际码头）与算法为其分配的驳船卸货港不一致时，就需要在宁波港内通过卡车/拖车进行转码头，产生额外费用。

实际案例（场景 8-17）：

箱 SEGU1983253：干线码头 = BLCTZS，驳船卸货港 = BLCT3 → 到港后从 BLCT3 用卡车拉到 BLCTZS

场景 8-17 各船明细：

船名	卸货港	卸箱数	转码头数	转码头率	费用
XIN YUAN HE 18	BLCT3	354	224	63.28%	67,846 元
XINGTONGHAI	BLCTMS	159	0	0.00%	0 元
HUA ZHENG JI YUN	BLCT3	290	26	8.97%	7,086 元
XIN YUAN HE 18 转码头率高达 63%，是因为该船被算法分配去 BLCT3，但其携带了大量目的港为 BLCT/BLCT2/BLCTZS 的货箱（共 224 箱需在宁波港内陆运转移）。这正是算法评分中 转码头率 负分项所对冲的——与积载率提升相权衡的代价。
