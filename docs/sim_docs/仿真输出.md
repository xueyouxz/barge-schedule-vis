## **🔗 排船算法输出与最终输出的关系总结**

### **1. 决策层关系**

```
排船算法输出
    ↓
决定了：
  - 船舶航线（哪些港口，什么顺序）
  - 集装箱分配方案（哪个箱上哪条船）
  - 是否中转、共舱、多靠
    ↓
影响：
  - 仿真事件的执行顺序
  - 装卸船的时间和地点
  - 运输路径和转运次数
```

### **2. 数据流关系**

```
算法输出（内存中）
├─ barge_route → barge["route"]
│   └─ 决定船舶访问哪些港口
│       └─ 影响 trace（轨迹记录）
│           └─ 输出到 barge_infos.json
│
├─ allocated_routes → container["route"]
│   └─ 决定集装箱运输路径
│       ├─ 影响 loading_unloading_records
│       │   └─ 输出到 results.xlsx
│       └─ 影响 container_records
│           └─ 输出到 container_records.csv
│
└─ 中转/转码头决策
    └─ 影响 barge_records（费用统计）
        └─ 输出到 metrics_by_nj_date.json
```

### **3. 指标溯源关系**

最终指标如何从算法输出计算得来：

| 最终指标 | 算法输出来源 | 计算方式 |
|---------|------------|---------|
| **积载率 97.14%** | `allocated_routes` | 分配的箱TEU总和 / 船最大TEU |
| **转码头率 17.65%** | `allocated_routes`中跨码头箱 | 目的地≠卸货港的箱数 / 总箱数 |
| **中转费用 27000元** | `route`中包含TC2的箱 | 中转箱数 × 单箱费用(600元) |
| **周转时间 42.5h** | `barge_route`的航线长度 | trace记录的时间差 |
| **太仓占用 8.5h** | `route`中转的装卸时间 | 中转箱数 × 单箱装卸时间 |

### **4. 举例说明**

**算法决策：**
```python
箱NBYU0042179分配到路径：[("LTA", "TC2", 0), ("TC2", "BLCT3", 2)]
                        ↓
表示：船0从LTA装载，TC2卸货；船2从TC2装载，BLCT3卸货
```

**最终体现：**
1. **装卸记录**：
   - `LTA_装船.xlsx`: 2025-08-16 18:36 装箱NBYU
   - `TC2_卸船.xlsx`: 2025-08-17 11:06 卸箱NBYU（船0）
   - `TC2_装船.xlsx`: 2025-08-17 15:36 装箱NBYU（船2）
   - `BLCT3_卸船.xlsx`: 2025-08-18 08:42 卸箱NBYU

2. **费用统计**：
   - 中转费用：+600元（1小箱×600）
   - 转码头费用：+150元（BLCT3→BLCT）

3. **指标贡献**：
   - 积载率：+1/70 = +1.43%
   - 中转率：+1箱
   - 太仓占用：+0.5h（装卸时间）

---


**数据流：**
```
算法决策 → 事件驱动仿真 → 过程记录 → 指标计算 → 可视化输出
  (方案)     (执行模拟)    (时序数据)   (统计分析)  (Excel/JSON)
```
